01 — The Problem

Urban Congestion
Demands Smart Response

Traditional traffic signal systems operate on fixed timing cycles, completely blind to real-world traffic density. During rush hours, this causes massive queues, fuel waste, and emergency vehicle delays.

Urban populations are growing rapidly. India alone loses billions of rupees annually due to traffic congestion — reduced productivity, increased fuel consumption, and harmful emissions.

The solution demands a system that sees, thinks, and adapts — using computer vision and distributed decision-making to dynamically manage signal timings based on live traffic data.

04 — PCA Pipeline

7-Stage
Processing Workflow

01

CCTV Frame Capture

Live video feeds from intersection cameras are captured continuously. Frames are extracted at set intervals to balance accuracy with processing speed.

OpenCV
02

Convert to Grayscale

RGB color images are converted to single-channel grayscale to reduce data dimensionality and focus on structural features rather than color.

OpenCV
03

Resize Image

Images are standardized to a uniform resolution, ensuring consistent input dimensions for the PCA model across all camera feeds.

Preprocessing
04

Flatten to 1D Vector

The 2D pixel matrix is flattened into a single 1D vector array, transforming image data into a format suitable for PCA analysis.

NumPy
05

Apply PCA

Principal Component Analysis is applied to decompose the flattened vectors, reducing high-dimensional image data into its most informative components.

Scikit-learn
06

Extract Dominant Features

Principal components that capture the most variance are selected. These represent the structural patterns most significant for vehicle detection.

Feature Extraction
07

Estimate Traffic Density

Extracted features are mapped to traffic density scores, which are fed to the DCSP decision engine for optimal signal timing adjustment.

DCSP Input
